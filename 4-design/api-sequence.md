# Sequence-диаграмма API-взаимодействия

> Этап 3. Взаимодействие Клиент ↔ Бэкенд.

## Сценарий: Успешная запись (UC-1)

1. **GET /slots?start_date=...&end_date=...**
   * Клиент запрашивает расписание на 7 дней.
   * Бэкенд возвращает список объектов `Slot`.
   
2. **GET /slots/{slot_id}**
   * (Опционально) Клиент открывает детальную карточку конкретного слота.
   * Бэкенд возвращает актуальные `available_places`.

3. **POST /bookings**
   * Клиент отправляет тело брони: `slot_id`, `client_name`, `phone`, `needs_rental`.
   * **Важно:** Заголовок `Idempotency-Key: <UUID>`.
   
4. **Response (201 Created)**
   * Бэкенд подтверждает запись и возвращает `booking_id`.
   * Локальное состояние клиента обновляется (места в слоте уменьшаются).
