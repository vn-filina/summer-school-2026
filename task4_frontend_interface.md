# Задача 4. Разработка клиентского интерфейса (Streamlit, интеграция с REST API и управление состоянием сессии)

### 1. Симптом / Цель
Бэкенд-прототип успешно разработан и протестирован. Для обеспечения полноценного пользовательского опыта необходим отзывчивый, современный графический интерфейс. Цель — совместно с ИИ-ассистентом разработать фронтенд-приложение на Streamlit, настроить сетевую интеграцию с бэкендом через HTTP-запросы и реализовать бесшовный личный кабинет клиента.

### 2. Требования
Разрабатываемый интерфейс должен визуализировать и строго соблюдать заложенные Use Cases и функциональные требования:
* **Управление состоянием:** Использование `st.session_state` для безопасного хранения токена авторизации (`token`) и имени пользователя между перезапусками страниц фронтенда.
* **Динамический расчет (FR-006):** Интерактивный пересчет итоговой стоимости на форме записи при переключении галочки «Нужен прокат».
* **Защита сетевых запросов:** Генерация уникального UUID на стороне фронтенда для отправки в заголовке `Idempotency-Key` при каждом бронировании.
* **Реактивность (UC-2):** Мгновенное обновление статусов броней в личном кабинете при нажатии кнопки отмены.

### 3. Реализация и доработка
Вместе с ИИ я спроектировала и запрограммировала интерфейс в файле `app.py`, разделив его на три ключевых экрана с помощью условного рендеринга:

1. **Экран авторизации:** Если `st.session_state.token` пуст, клиенту рендерится форма входа. Фронтенд собирает данные, отправляет их на бэкенд `/auth/verify-code` и, получая UUID пользователя в качестве токена, сохраняет его в сессию, скрывая экран входа.
2. **Экран расписания (Главная):** Фронтенд выполняет запрос `GET /slots`. Полученный JSON-массив преобразуется в интерактивные карточки мастер-классов. Реализована блокировка кнопки «Записаться», если количество свободных мест `available_places <= 0`.
3. **Личный кабинет («Мои записи»):** Запрашивает данные с `/bookings/me` с передачей Bearer-токена в заголовках. Отображает карточки бронирований с понятными статусами (🟢 Активна, ⚪ Отменено, 🔴 Отменено (поздняя)). Здесь же выведена кнопка «Отменить запись», посылающая сигнал на бэкенд и вызывающая `st.rerun()` для моментального обновления UI.

### 4. Отправленные ИИ промпты (Мои запросы к ассистенту)
> **Мой запрос 1:** «Теперь давай напишем фронтенд на Streamlit в файле app.py. Начнем с экрана авторизации. Сделай форму ввода телефона, имени и СМС-кода. Настрой st.session_state так, чтобы после успешного POST-запроса на /auth/verify-code access_token сохранялся в состоянии сессии, и приложение автоматически переключалось на основной интерфейс.»
> 
> **Мой запрос 2:** «Теперь давай сделаем вывод расписания. Сделай GET-запрос на наш бэкенд /slots, вытащи оттуда список занятий и красиво отрисуй их в виде карточек. Выведи название программы, имя мастера, дату, время и остаток мест. Если мест больше нет, сделай кнопку "Записаться" неактивной (disabled).»
> 
> **Мой запрос 3:** «Внедрить в форму бронирования чекбокс "Нужен прокат инструментов и фартука (+300 руб.)". Сделай так, чтобы итоговая цена на экране пересчитывалась на лету в зависимости от выбранной программы и галочки проката. А перед самой отправкой POST /bookings сгенерируй uuid.uuid4() и передай его в Header как Idempotency-Key.»
> 
> **Мой запрос 4:** «Добавь вкладку "Мои записи". Она должна делать запрос на /bookings/me с токеном авторизации из session_state. Покажи пользователю все его брони с красивыми цветными статусами. Если бронь в статусе active, добавь кнопку "Отменить запись", которая шлет POST на эндпоинт отмены бэкенда и сразу вызывает st.rerun(), чтобы статус обновился мгновенно.»

### 5. Ручная проверка (Тестирование)
Я проверила интеграцию фронтенда и бэкенда на реальных сценариях:
1. Запустила бэкенд и фронтенд. 
2. Прошла авторизацию с тестовым кодом `1234`. Экран входа успешно скрылся, система поприветствовала меня по имени.
3. Нашла в расписании слот по ручной лепке (1500 руб.), поставила галочку проката — цена на экране мгновенно изменилась на 1800 руб. Нажала «Записаться». Система выдала зеленый виджет `st.success`.
4. Перешла во вкладку «Мои записи» — созданное бронирование успешно отобразилось со статусом «🟢 Активна». Связка фронтенд-бэкенд работает стабильно.
