# Задача 4. Исправление бага №1: Ошибка расчета итоговой стоимости проката

### 1. Симптом / Цель
При создании бронирования с включенной опцией «Нужен прокат инструментов и фартука» (`needs_rental=True`), итоговая стоимость `final_price` в базе данных сохранялась неверно. Система записывала только базовую стоимость самого мастер-класса, полностью игнорируя стоимость аренды инвентаря. Цель — обеспечить автоматический и корректный расчет финальной стоимости на стороне бэкенда перед сохранением в БД.

### 2. Требования
* **FR-006:** Автоматический расчет итоговой суммы (цена программы + прокат) до подтверждения бронирования.
* **Бизнес-ограничение:** Стоимость проката фиксирована в коде и составляет `RENTAL_PRICE = 300` рублей.

### 3. Реализация и доработка
В файле `crud.py` внутри функции `create_booking` я добавила условный расчет стоимости. Теперь перед коммитом в базу данных переменная `final_price` динамически рассчитывается с учетом статуса флага аренды:

```python
final_price = base_price + (RENTAL_PRICE if req.needs_rental else 0)
```
### 4. Отправленные ИИ промпты
> **Промпт 1:** «В функции create_booking бэкенд сохраняет final_price равным просто цене слота, забывая проверить флаг req.needs_rental. Исправь на если флаг True, к итоговой цене прибавлялась константа RENTAL_PRICE (300 рублей).»

Я запустила бэкенд и веб-интерфейс, после чего проверила сценарии:

3. **Тест-кейс 3:** Активировала чекбокс "Нужен прокат фартука и инструментов".
4. **Тест-кейс 4:** Нажала кнопку "Записаться".
5. **Тест-кейс 5:** Перешла в личный кабинет в раздел "Мои записи". Проверила итоговую сумму: на экране и в базе данных SQLite в таблице bookings для поля final_price зафиксировалось корректное значение 1800 руб. вместо прежних 1500. Тест успешно пройден.
