# Функциональные требования

> Этап 2. Что система должна делать (функции, поведение).

## Функциональные требования (MVP)

| ID | Требование                                                                                                   | Приоритет | Источник |
| :-- |:-------------------------------------------------------------------------------------------------------------| :-- | :-- |
| **FR-001** | **Вывод расписания.** Система отображает список слотов на 7 дней вперед.                                     | Must | R-027 |
| **FR-002** | **Детали слота.** В карточке отображать: программу, инструктора, дату, время, статус, кол-во свободных мест. | Must | Бриф |
| **FR-003** | **Идентификация.** Запись осуществляется по Имени и Телефонному номеру (валидация формата).                  | Must | Ответ №6 |
| **FR-004** | **Блокировка овербукинга.** Запрет записи, если свободных мест <= 0.                                         | Must | BR-1 |
| **FR-005** | **Учет проката.** В форме записи есть опция «Нужен прокат» (инструменты/фартук).                             | Should | Бриф |
| **FR-006** | **Расчет цены.** Авто-расчет итоговой суммы (цена программы + прокат) до подтверждения.                      | Should | Ответ №10 |
| **FR-007** | **Отмена (ранняя).** Если до начала $\ge 3$ч: статус «Отменено», место возвращается в слот.                  | Must | R-002 |
| **FR-008** | **Отмена (поздняя).** Если до начала $< 3$ч: статус «Отменено (поздняя)», место НЕ возвращается.             | Must | R-002 |
| **FR-009** | **Отмена мастерской.** При отмене слота организатором: статус «Отменено клубом», повторная запись закрыта.   | Must | R-008 |
| **FR-010** | **Empty State.** Если расписания нет — отображать сообщение «На ближайшие дни мест нет».                     | Should | R-027 |

---

> **Вне скоупа (Phase 2):** Оценки инструкторов, онлайн-оплата, программа лояльности и функции администратора (создание расписания) обеспечиваются существующей инфраструктурой и в рамках клиентского приложения не реализуются.
