# Задача 4. Исправление бага №3: Некорректный расчет правила 3 часов при отмене (Timezone Mismatch)

### 1. Симптом / Цель
При попытке отменить бронирование на занятие, до которого осталось более 4 часов, система ошибочно присваивает статус `late_cancelled` (Поздняя отмена) вместо `cancelled` (Ранняя отмена). Это происходит из-за того, что в `crud.py` при проверках берется локальное время `datetime.now()`, в то время как слоты в базе данных могут быть записаны в UTC (или наоборот, в зависимости от окружения/Docker-контейнера сервера).

### 2. Требования
* **FR-007 / FR-008:** Если до начала занятия $\ge 3$ часов — статус «Отменено» (место возвращается). Если $< 3$ часов — статус «Поздняя отмена» (место НЕ возвращается).

### 3. Реализация и доработка
В файле `crud.py` внутри функции отмены бронирования (или в месте проверки таймингов) расчет времени приведен к единому стандарту. Так как все слоты генерируются через наивное время или UTC, мы синхронизировали вызов `now`.

В функции отмены бронирования в `crud.py`:
```python
# Было: разница проверялась некорректно из-за разного формата времени сервера
# if booking.slot.start_time - datetime.now() ...

# Стало (синхронизация вызова времени с форматом хранения слотов):
# Приводим к одному виду без учета таймзон (чтобы избежать сдвигов на 3 часа)
now = datetime.now().replace(tzinfo=None)
start = booking.slot.start_time.replace(tzinfo=None)

time_diff = start - now

# FR-007 и FR-008: Граница в 3 часа (3 * 3600 секунд)
if time_diff.total_seconds() >= 3 * 3600:
    booking.status = db.BookingStatus.cancelled
    booking.slot.available_places += 1  # Ранняя отмена: место возвращается
else:
    booking.status = db.BookingStatus.late_cancelled
    # Поздняя отмена: место НЕ возвращается, счетчик не трогаем
```
### 4. Отправленные ИИ промпты
> **Промпт 1:** «При отмене занятия разница во времени считается неверно, потому что datetime.now() возвращает локальное время тачки, а в базе слоты лежат в другом формате, из-за чего возникает сдвиг на 3 часа и отмена всегда считается поздней. Помоги очистить таймзоны перед вычитанием дат.»
